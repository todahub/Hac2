## AI概要

### 全体構成
- **表示層**: カエルの人形、楽器保持パーツ、口の開閉などの視覚ギミック。
- **制御層**: ホスト機Arduinoがテンポと拍を管理する。
- **演奏層**: クライアント機Arduinoと Processing が音を生成・再生する。
- **通信層**: ホスト機から各クライアントへ同一データをブロードキャストする。

### ハードウェア役割
- ホスト機: BPM算出、開始停止、拍の送信。
- クライアント機: 拍の受信、開始タイミング制御、音の再生。
- 表示機構: LED、サーボ、カエル造形で状態を見せる。

### ソフトウェア役割
- ホスト側は `main` と `config` を分け、通信設定やピン配置を集中管理する。
- クライアント側も `main` と `config` を分け、個体IDや追いかけ拍数を切り替えやすくする。
- Processing 側は波形生成と再生制御を分離する。

### 処理フロー
1. ホスト機が可変抵抗から BPM を読む。
2. 拍情報をブロードキャストする。
3. クライアント機が受信して自分の開始タイミングを判定する。
4. Processing で音を生成し、Arduino 経由で再生する。
